# sdd-sync（文档合并）

## 定位

将 change 中的设计变更智能合并回 `.ai/doc/` 主文档。

## 前置检查

1. 无活跃 change 且用户未指定 change → 拒绝
2. plan.md 为空（仅占位）→ 警告"plan.md 无设计内容，无法同步"，终止
3. plan.md 的 `## 文档变更` 表格为空或不存在 → 提示"本次 change 未声明文档变更，无需同步"，终止

## 执行流程

### 步骤 1：分析变更

1. 读取 change 的 proposal.md 和 plan.md
2. 从 plan.md 的 `## 文档变更` 表格提取 doc 变更清单
3. 从 plan.md 的 `## 04 更新计划`（如有）提取 04 增删清单
4. 按目标文件分组

### 步骤 2：并行冲突检测

1. 扫描 `.ai/changes/`（排除 `.ai/changes/archive/`）中所有其他活跃 change 的 plan.md
2. 检查它们的 `## 文档变更` 表是否声明了相同目标文件 + 相同章节
3. 如有冲突 → AskUserQuestion（继续合并 / 跳过该文件 / 取消）

### 步骤 3：逐文件智能合并

对每个受影响的 doc 文件：

0. **创建备份**：
   - 在 change 目录下创建 `.sync-backup/` 目录
   - 将本次所有将被修改的 doc 文件原样复制到 `.sync-backup/`（保留原文件名）
1. **重读全文**（强制，不可跳过），理解其结构和当前内容
2. 读取 change 中的相关设计内容
3. **按操作类型执行合并**：

   | 操作 | 合并规则 |
   |------|----------|
   | **ADDED** | 1) 确定插入位置（寻找最相关的已有章节附近）2) 如果是新增模块，在 `.ai/doc/03-详细设计/` 下创建新文件（基于 `_模板.md`）3) 如果目标文件不存在，创建新文件写入变更内容作为初始版本 |
   | **MODIFIED** | 1) 定位到具体章节 2) **局部替换**涉及的内容 3) **保留**未提及的章节 4) 不做全文件替换 |
   | **DELETED** | 1) AskUserQuestion 让用户选择："标记废弃"还是"直接删除" 2) 如选择标记废弃：定位到具体章节，标记为 `> **已废弃**（来自 change NNN）`，保留历史痕迹 3) 如选择直接删除：移除该章节内容 |

4. **质量自检**（4 项）：
   - [ ] 没在末尾追加（创建心态）
   - [ ] 内容属于该文件的角色（对照文件角色表）
   - [ ] 没有"讨论了A和B选了A"（不放实现过程）
   - [ ] 变更摘要与 plan.md 的声明一致

5. **尊重文件角色**：

   | 文件 | 角色 | 不该出现的内容 |
   |------|------|--------------|
   | `00-架构.md` | 模块划分、依赖、数据流 | 需求细节、API 契约、技术债务 |
   | `01-需求.md` | 做什么和为什么 | 实现细节、数据域、行为约束 |
   | `02-技术选型.md` | 选型理由 | 需求描述、API 细节、反模式 |
   | `03-详细设计/xxx.md` | 该模块的详细设计 | 需求动机、全局架构、模块陷阱 |
   | `04-问题与改进.md` | 当前有效问题汇总 | 需求描述、设计理性、实现细节 |

6. **处理 04 更新计划**（如 plan.md 包含 `## 04 更新计划`）：
   
   **删除已解决的问题**：
   - 遍历"本次将解决的问题"列表
   - 在 `04-问题与改进.md` 中找到对应编号条目，**直接删除**
   - 不保留已修复条目（历史见 git）
   
   **添加新引入的问题**：
   - "取舍代价" → 以 TR-XXX 编号写入 04 技术层 > 设计取舍代价
   - "新识别问题" → 以相应编号（MT/FD/AP 等）写入 04 对应层级
   - 确保编号不与已有条目冲突（取最大编号+1）

### 步骤 4：确认与记录

1. 展示变更摘要（哪些文件改了什么）
2. 更新 `.ai/project-log.md` 的活跃 Change 区段

## 联动设计

- **拒绝路径**：
  - 无活跃 change → 提示"请先通过 `/sdd-propose` 创建 change"
  - plan.md 为空 → 提示"请先通过 `/sdd-explore` 制定方案"
  - 文档变更表为空 → 提示"无需同步，可直接运行 `/sdd-archive`"
- **完成后引导**：`✅ 文档已同步。接下来可通过 /sdd-archive 归档本次 change。`
- **如无需同步**：`ℹ️ 本次 change 未涉及文档变更，无需同步。可直接运行 /sdd-archive 归档。`
