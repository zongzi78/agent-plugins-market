# 阶段 6：集中诊断

**目标**：基于完整的 00-03 文档和观察便签，系统性发现所有层级的问题，一次性填充 04 文档。

> **本 SOP 的上下文**：
> - 文件角色定义 → `../SKILL.md`「2. 文件角色对照表」
> - 质量规则（正向/负向） → `../SKILL.md`「1. 通用规则」
> - 产出模板路径见 SKILL.md「各阶段资源映射」表
> - 全局约束：代码证据使用符号锚点格式（ClassName::methodName() 或 filename.cpp -> functionName()）

## 执行步骤

1. **调用 ai-sdd-diagnose**：`diagnose(layer=ALL, depth=L2)`
   - 该 Skill 读取 00-03 全部文档（物化的系统理解）
   - 读取 project-log.md 中的「诊断观察便签」区段（阶段 2-5 的洞察线索）
   - 执行全部诊断范畴的搜索清单
   - 从架构层→需求层→技术层→设计层，逐层梳理
   - 关联跨层级的根因和表现
   - 全局标准化严重程度
   - 去重、合并
   - 写入 `.ai/doc/04-问题与改进.md`
   - 删除 project-log.md 中的「诊断观察便签」区段

2. **输出诊断摘要**供人类确认：
   ```
   📊 诊断结果摘要：
   - 架构层：AV-X 处、DD-X 处
   - 需求层：DR-X 处、FD-X 处
   - 技术层：VR-X 处
   - 设计层：MT-X 处、CD-X 处
   - 合计：X 条有效问题
   ```

## 产出

- `.ai/doc/04-问题与改进.md`（完整填充）
- project-log.md「诊断观察便签」区段已删除

## 确认清单

产出后暂停，输出以下确认清单：

```
📋 阶段 6 确认清单 — 集中诊断

请逐一检查以下内容：

□ 问题列表是否完整？有无遗漏的重要问题？
□ 各问题的严重程度（高/中/低）是否合理？
□ 问题描述和修复建议是否准确？
□ 是否有问题需要补充更多上下文？

快捷回复：
  [全面] — 确认问题列表完整且准确
  [补充X] — 追加遗漏的问题
  [修改Y] — 修正某问题的描述/严重程度/修复建议
  [删除Z] — 删除某条误报
```

## 人类确认后的强制更新步骤

收到人类回复后：
1. 按人类回复修改 04 文档（修改/追加/删除问题条目）
2. 更新 front matter：`last_updated`（今天日期）、`status`（`reviewed`）
3. 在 project-log.md 追加记录：`[日期] 阶段6 经人类确认`

## 人类回复后分支处理

| 人类反馈 | 你的操作 |
|---------|---------|
| 确认全面 | 执行上方「强制更新步骤」 |
| 补充新问题 | 追加到 04 对应层级，重新编号（如有必要），执行「强制更新步骤」 |
| 修改某问题 | 按指示修改，执行「强制更新步骤」 |
| 删除某问题 | 从 04 中删除对应条目，执行「强制更新步骤」 |
| 人类未回复/跳过 | 内容保持不变；继续下一阶段 |
