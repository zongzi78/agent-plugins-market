# 阶段 4：技术栈确认

**目标**：记录当前技术选型。这是"现状盘点"，不猜测选型动机。

> ⚠️ 执行前提：本阶段涉及 confidence 标记。开始前必须先 Read `skills/ai-sdd-reverse/术语与概念体系.md`，确认理解：
> - confidence 三级定义：`confirmed`（已确认）= 人类确认过 / `inferred`（推断）= 有代码证据但未确认 / `speculative`（推测）= 证据不足
> - front matter 的 `confidence` 字段取整体最高规则：全部 confirmed → `confirmed`；存在 speculative → `speculative`；否则 `inferred`
> - `status` 字段：人类确认后 → `reviewed`；全部 confirmed → `baseline`

## 执行步骤

1. **分析依赖**：从包管理器文件中提取所有直接依赖项及其版本
2. **版本漂移评估**：每个关键依赖对比当前版本 vs 包管理器可获取的最新稳定版，标注差距和迁移复杂度
3. **使用模式分析**：每个技术组件标注是标准用法还是定制/非标准用法
4. **构建/测试/部署验证**：从实际配置文件中提取工具链信息，不假设标准工具
5. **技术债务识别**：过时依赖、不一致编码模式、硬编码配置、缺乏测试的模块
6. **提取"缺失决策理由"列表**：列出从代码中无法推断但值得记录的技术选择（如"为什么选 PostgreSQL 而不是 MySQL？代码无法推断"）
7. **技术约束与反模式**：记录代码中隐含的技术禁止事项和已出现的反模式

## 产出

`.ai/doc/02-技术选型.md`（基于模板生成）

## 产出后自检

生成完成后立即逐项核对：
- [ ] 所有直接依赖及版本已列出，含证据来源
- [ ] 每个关键依赖标注了使用模式（标准用法 / 非标准用法）
- [ ] 构建/测试/部署配置已验证实际文件路径
- [ ] 版本漂移分析表已填写
- [ ] 缺失决策理由列表已填写
- [ ] **置信度统计**（必须在输出确认清单前完成）：
  a. 逐条列出文档中所有可信度标记（编号 + 当前等级）
  b. 汇总：已确认 N 处、推断 M 处、推测 K 处
  c. 代入规则：存在推测 → `speculative`；全部已确认 → `confirmed`；否则 → `inferred`
  d. 将结果写入 front matter 的 `confidence` 字段

## 关于可信度

本阶段产出的信息分为两类：
- **直接可验证的事实**（如依赖名称、版本号、配置文件路径）→ 直接标记为 `已确认`
- **推断性内容**（如使用模式评估、版本迁移建议、反模式归类）→ 标记为 `推断`

## 确认清单

产出后暂停，输出以下确认清单：

```
📋 阶段 4 确认清单 — 技术栈确认

□ 技术栈总览：所有主要技术是否都识别到了？有无遗漏或误判？
□ 关键依赖：版本号是否正确？使用模式（标准/非标准）判断是否合理？
□ 版本漂移：需要特别关注的版本差距是否有遗漏？
□ 构建/测试/部署：工具链识别是否准确？

如有纠正请直接说明（如"Redis 版本是 6.2 不是 7.0"、"漏了 Nginx"、"XX 不是非标准用法"）。
如需跳过此确认，回复"跳过"即可继续。
```

## 人类确认后的强制更新步骤

收到人类回复后，在执行分支处理之前，**必须**按顺序完成：

1. **逐条更新文档内容**：遍历文档中所有标记为 `推断` 的声明——人类确认的 → 改为 `已确认`
2. **重新统计置信度**：
   a. 重新遍历所有可信度标记（因第1步可能改变了标记）
   b. 输出统计：已确认 N 处、推断 M 处、推测 K 处
   c. 代入规则：存在推测 → `speculative`；全部已确认 → `confirmed`；否则 → `inferred`
   d. 更新 front matter：`confidence`、`last_updated`（今天日期）、`status`（`reviewed` 如果人类确认）
3. **在 project-log.md 追加记录**：`[日期] 阶段4 经人类确认，N 处推断升级为已确认`

只有完成以上步骤后，才进入阶段 5。

## 人类回复后分支处理

| 人类反馈 | 你的操作 |
|---------|---------|
| 确认全部正确 / 回复"跳过" | 执行上方「强制更新步骤」后继续 |
| 某信息需要纠正 | 按人类指示直接修改文档；修改后的内容标记为 `已确认`；然后执行「强制更新步骤」 |
| 补充了新信息 | 追加到对应章节，标记为 `已确认`；然后执行「强制更新步骤」 |
| 人类未回复 | 所有内容保持原可信度；继续下一阶段 |
